## License And Provenance Summary

The World English Bible project describes WEB as public domain and not copyrighted. eBible.org's WEB pages also state that "World English Bible" is a trademark of eBible.org.

Practical implications for WCM:

- WCM may store and display faithful copies of the WEB text if the source/license review is approved.
- WCM must not call a modified text "World English Bible".
- If WCM imports WEB, it should preserve source/provenance metadata and identify the edition accurately.
- If WCM normalizes punctuation, removes headings, or changes formatting for runtime display, the import report should distinguish formatting normalization from textual alteration.

The candidate WEBP detail page identifies:

```

The same page reports:

```

The Classic WEB page reports HTML generated by eBible.org from source files dated 2026-06-11, but WEBP's own package update metadata must be verified from the exact selected package before import.

## Format Availability

eBible.org lists several available formats for `engwebp`.

Most relevant for import:

1. `engwebp_usfm.zip`
   - Best first candidate for WCM import.
   - USFM is a standard Scripture source format.
   - Expected to preserve book/chapter/verse markers well enough for importer normalization.

2. `engwebp_usfx.zip`
   - XML-like Scripture interchange format.
   - Useful alternative if USFM parsing proves harder than expected.

3. `engwebp_readaloud.zip`
   - Plain text canon-only chapter files.
   - Potentially easy to inspect, but may be less explicit for precise verse parsing depending on file structure.

Formats not recommended as first import source:

- HTML ZIP: good for reading/offline display, not ideal as authoritative import input.
- PDF/ePub/Kindle: display formats, not ideal for structured import.
- VPL + SQL: includes SQL and should not be the first choice because WCM policy avoids SQL dump-based import for Scripture data movement.
- Crosswire Sword module: useful for distribution, but adds module-specific parsing concerns.

## Risks

### Source Variant Confusion

WEB, WEBU, WEBP, WEBBE, WEBPB, WEBC, WMB, and related variants are distinct enough that WCM must not use a generic `WEB` label internally without recording the exact source edition.

Recommendation:

```txt
Use frontend/public version code WEB only if metadata clearly records source edition WEBP / ENGWEBP.
```

### Trademark Boundary

The text is public domain, but the "World English Bible" name is trademarked. WCM should identify faithful imported copies accurately and avoid using the name for modified text.

### Format Parsing

USFM is a strong candidate but still requires parser validation for:

- book identifiers.
- chapter markers.
- verse markers.
- poetry/paragraph markers.
- footnotes.
- headings.
- punctuation and quotation style.

### Verse Count Differences

WEBP should be a 66-book Bible, but the selected package still needs count validation against WCM's canonical book/chapter/verse model.

### Accidental Generated Data Commit

Downloaded ZIP files, extracted USFM/USFX/plain text files, generated JSON, SQL, or import packages must remain untracked and ignored.

### Production Write Risk

No WEB import should target staging or production until local dry-run and local apply have both passed and a promotion plan is separately approved.
